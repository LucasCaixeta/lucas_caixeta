# Contributing to lucascaixeta.com

Thank you for your interest in this project! This document provides guidelines for contributing.

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on the code, not the person

## Development Setup

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Run tests
npm test

# Format code
npm run format

# Build for production
npm run build
```

## Making Changes

1. Keep changes focused and minimal
2. Follow the existing code style
3. Add tests for new functionality
4. Update documentation as needed
5. Write meaningful commit messages

## Code Style

- Use 2-space indentation
- Use semicolons
- Use single quotes for strings
- Use meaningful variable names
- Add comments for complex logic

## Testing

- Run tests before submitting changes: `npm test`
- Write tests that cover your changes
- Test on multiple browsers when possible

## Commit Messages

Follow conventional commit format:
- `feat: Add new feature`
- `fix: Fix a bug`
- `docs: Update documentation`
- `style: Code formatting changes`
- `test: Add or update tests`
- `refactor: Refactor code structure`
- `perf: Improve performance`

## Pull Requests

Include:
- Clear description of changes
- Link to any related issues
- Screenshots for UI changes
- Test results

## Questions

For questions or suggestions, please open an issue or contact lucas directly.

Thank you for contributing!
